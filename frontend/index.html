SEGEL — HOW IT WORKS

Three on-chain steps. Two ZK circuits. Zero leaked bids.

Step 01 · Post RFQ

A maker posts a request for quote — pair, size band, price range and a deadline — stored in the Soroban contract.

✓Pair & size band
✓Min / max price range
✓On-chain deadline
✓USDC escrow opens
Step 02 · Sealed Bid

Takers commit Poseidon(bid, nonce, addr). A range proof shows min ≤ bid ≤ max — invalid bids die on arrival, the amount never leaks.

✓Poseidon commitment
✓In-browser proving
✓Proof-of-funds + allowlist
✓Bid stays hidden
Step 03 · ZK Settle

One proof reveals only the winner and the clearing price. The second-highest bid sets the price (Vickrey), computed entirely in-circuit; losing bids never appear.

✓Winner + clearing price only
✓Vickrey second-price, in-circuit
✓Commits bind to on-chain set
✓Atomic escrow release
